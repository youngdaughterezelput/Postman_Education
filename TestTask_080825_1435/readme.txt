Файл .env должен содержать, так как класс не предусматривает создание dotenv

BASE_URL=https://your-api-base-url.com
ORGANIZATION_ID={{}}
AUTH_TOKEN={{bearer_token}}

необходимо установить библиотеку:
pip install python-dotenv

Запуск теста из терминала:
python optimization_api_tests.py

Объяснение кода:

setUpClass: 
Инициализирует общие данные для всех тестов (URL, ID, токен) один раз. Использует python-dotenv для безопасной загрузки из .env.

setUp: 
Выполняет HTTP-запрос к API один раз перед запуском тестов (кроме негативных). Результат сохраняется в self.response, чтобы не делать запрос в каждом тесте.

test_XX_...: 
Каждый метод, начинающийся с test_, является отдельным тестом. Они используют unittest методы assert... для проверки условий.

assertEqual, assertIn, assertIsInstance: 
Методы unittest для проверки равенства, наличия элемента, типа данных и т.д.

test_99_status_401_unauthorized: 
Пример негативного теста, который делает отдельный запрос без авторизации.

if __name__ == '__main__':: 
Позволяет запустить файл напрямую как скрипт для выполнения тестов.
